Documentação · v1.x

Arcana DataTable

Uma datatable tipada para Vue 3 e React. A mesma configuração, o mesmo comportamento, o mesmo resultado — em qualquer um dos dois frameworks.

Começando

Instalação

A biblioteca é distribuída como um único pacote npm. Os adaptadores de React e Vue são expostos em subcaminhos separados (@arcanalabs/datatable/react e @arcanalabs/datatable/vue), então o bundle final só carrega o framework que você usa.

Instale com o gerenciador de pacotes da sua preferência e pronto — não há dependências transitivas de UI.

Primeiro uso

Todo o comportamento da tabela vive em um único objeto config. Você declara as colunas, aponta a origem dos dados e renderiza o componente ArcanaDataTable — busca, ordenação e paginação já funcionam sem nenhuma linha extra.

O exemplo ao lado usa o modo remote: a cada interação do usuário, a tabela chama seu datasource com os parâmetros atuais.

Estilos

A folha de estilos é opt-in: importe @arcanalabs/datatable/styles.css uma única vez, na raiz da aplicação. Ela cobre a tabela, os filtros, a paginação e os estados de carregamento.

Se preferir um visual próprio, basta não importar o CSS — a marcação é semântica e estável, pensada para ser estilizada por fora.

Conceitos

Modos remote/dataset

Existem duas formas de alimentar a tabela, e essa escolha define quem faz o trabalho de filtrar, ordenar e paginar:

  • remote — cada filtro, ordenação ou troca de página dispara uma chamada ao datasource(params) (ou à url configurada). A resposta esperada tem o formato { rows, total, page }.
  • dataset — você entrega a coleção completa em memória e a tabela resolve tudo localmente, com zero requisições.

O mode é inferido a partir do que você fornece; declare-o apenas quando quiser ser explícito.

Colunas

Cada coluna é um objeto DataTableColumn com um name (a chave na linha) e um label (o texto do cabeçalho). Campos como type controlam a formatação — por exemplo, CURRENCY renderiza valores monetários — e searchType define o controle de busca da coluna.

A propriedade columns também aceita uma função () => DataTableColumn[], útil quando as colunas dependem de permissões ou de estado da aplicação.

Paginação

A paginação vem habilitada por padrão com rowsPerPage: 10. No modo remote, a página atual e o tamanho da página viajam nos params enviados ao datasource; o total retornado pela resposta alimenta o contador do rodapé.

Para navegar programaticamente, use controller.paginate(page, size).

Recursos

Filtros

A busca por coluna vem ativa por padrão (searchEnabled: true). Cada coluna com um searchType ganha o controle adequado no cabeçalho; no modo remote, os valores preenchidos são enviados dentro dos params.

Use initialFilters para aplicar filtros já na primeira requisição, e setFilter / setFilters no controller para alterá-los em resposta a ações da sua aplicação.

Tipos de busca

O searchType de cada coluna define o controle exibido para o usuário. São sete tipos:

searchTypeControle
DATESeletor de data única
DATE_MONTHSeletor de mês/ano (competência)
DATE_RANGEIntervalo entre duas datas
BOOLEANAlternador sim/não
LISTLista fixa de opções
REMOTEOpções carregadas de um endpoint
COMPONENTComponente customizado seu

Ordenação

A ordenação por coluna vem habilitada por padrão (orderByEnabled: true): clicar no cabeçalho alterna entre ascendente e descendente. No modo remote, a ordenação atual segue junto nos params; no modo dataset, é resolvida em memória.

Para aplicar uma ordenação por código, use controller.applyOrderBy(orderBy) — a tabela atualiza os indicadores do cabeçalho e refaz a consulta.

Seleção múltipla

Com checkboxEnabled: true, cada linha ganha um checkbox e o cabeçalho permite selecionar a página inteira. A seleção sobrevive à troca de página.

Leia as linhas marcadas com getCheckedRows() e limpe tudo com clearCheckedRows() — típico em ações em lote como "exportar selecionados".

Seleção única

Quando o fluxo pede exatamente um registro — escolher um cliente em um diálogo, por exemplo — habilite radioButtonSelectionEnabled: true. Cada linha exibe um radio button e marcar uma linha desmarca a anterior.

A linha escolhida também é lida via getCheckedRows(), que nesse modo retorna no máximo um item.

Totais

Com footerSummarizerEnabled: true, a tabela adiciona uma linha de rodapé com o total das colunas numéricas — colunas CURRENCY são somadas e formatadas como moeda.

O valor consolidado de uma coluna também fica disponível por código, via getSummarizedValue(column).

Responsividade

Em telas estreitas, escolha o comportamento com responsiveMode:

  • HORIZONTAL_OVERFLOW — mantém a estrutura de tabela e habilita rolagem horizontal.
  • VERTICAL_RECORD — cada linha vira um cartão vertical, com o rótulo da coluna ao lado de cada valor.

Em listas longas, stickyHeaderEnabled: true mantém o cabeçalho visível durante a rolagem.

Ações e eventos

A requisição inicial acontece na montagem (sendRequestOnMounted: true); desative-a quando a tabela depender de um filtro que o usuário ainda vai escolher. Falhas do datasource caem em onRequestError, onde você decide como notificar.

Para refletir mutações sem refazer a consulta — o clássico "editei uma linha no diálogo" — use as mutações pontuais do controller: addRow, updateRow e removeRow. Quando quiser recarregar tudo, refresh().

Hooks

O controller é a instância que expõe os métodos programáticos da tabela. Obtenha-o com uma ref apontada para o componente — o padrão idiomático de cada framework — e chame qualquer método a partir da sua UI.

A lista completa de métodos está em Métodos do controller.
